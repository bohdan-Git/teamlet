# Content Type Guides

Detailed guidelines for each content type Teamlet produces.

### Landing Page Copy
- **Hero:** One clear headline (benefit, not feature) + subheading + CTA
- **Social proof:** Testimonials, numbers, logos - if available
- **Features:** 3-5 benefits with short descriptions (problem → solution format)
- **CTA sections:** Clear action, remove doubt, create urgency (honestly)
- **FAQ:** Address real objections, not filler questions

**Framework:** Problem → Agitation → Solution → Proof → CTA

### UI Microcopy
- **Buttons:** Action verbs ("Save changes" not "Submit", "Create account" not "Sign up")
- **Empty states:** Helpful + encourage action ("No projects yet. Create your first one →")
- **Error messages:** What happened + how to fix it (never blame the user)
- **Loading states:** Set expectations ("Analyzing your data... this takes ~10 seconds")
- **Success messages:** Confirm + suggest next action ("Saved! Share with your team?")
- **Tooltips:** One sentence max, answer "what does this do?"
- **Onboarding:** Guide through value, don't explain features

### Email Templates
- **Welcome:** Reinforce the decision, one clear next step
- **Password reset:** Direct, no marketing - just the link
- **Transactional:** Clear subject, key info first, action button
- **Newsletter/Updates:** Value first, news second, one CTA

### Product Descriptions
- **App Store / Product Hunt:** Hook in first line, benefits over features, social proof
- **GitHub README intro:** What it does, who it's for, why it's different (3 sentences max)
- **Social media:** Platform-appropriate length and tone

### Translations & i18n
- Ask: which languages? what i18n framework (next-intl, react-i18next, etc.)?
- Translate meaning, not words - adapt idioms and cultural references
- Keep UI string keys consistent and descriptive
- Handle pluralization rules per language
- Flag strings that need cultural adaptation (dates, currencies, humor)
